# Fig.4 benchmark 公开方法与软件资料

更新日期：2026-07-16

> 科学性修订：最终主测试只给盲谱、元素集合和测量元数据，不给 84 个生成 CIF，

## 先说明本机安装位置

本轮没有向 Conda `base` 环境安装任何包。当前 `base` 位于 `/opt/anaconda3`，以下环境均隔离在项目目录：

| 用途 | 类型 | 本地位置 |
|---|---|---|
| pymatgen 生成器原型 | Python 3.12 虚拟环境 | `fig4/benchmark/.venv/` |
| Dara | Python 3.12 虚拟环境 | `fig4/benchmark/method_envs/dara/` |
| XERUS 兼容性试装 | Python 3.12 虚拟环境 | `fig4/benchmark/method_envs/xerus_compat/` |
| GSAS-II + 可运行 XERUS | 官方 GSAS2MAIN Python 3.13/arm64 隔离环境 | `fig4/benchmark/method_envs/gsas2main/` |
| CrystalShift CIF 转换辅助 | Python 3.12 虚拟环境 | `fig4/benchmark/method_envs/crystalshift_python/` |
| CrystalShift + CrystalTree | Julia project | `fig4/benchmark/method_envs/crystalshift/` |
| Julia 包缓存 | 独立 Julia depot | `fig4/benchmark/.julia_depot/` |

这些 Python 环境的 `include-system-site-packages=false`，不会读取或污染 Conda base 的包。

## 公开方法

### 1. XERUS

- 类型：Python 包；GSAS-II Scriptable 的自动化封装。
- 文章：Pedro Baptista de Castro, Kensei Terashima, Miren Garbine Esparza Echevarria, Hiroyuki Takeya, Yoshihiko Takano, “XERUS: An Open-Source Tool for Quick XRD Phase Identification and Refinement Automation”.
- 年份与期刊：2022，*Advanced Theory and Simulations*。
- DOI：<https://doi.org/10.1002/adts.202100588>
- 代码：<https://github.com/pedrobcst/Xerus>
- 论文对应版本：<https://github.com/pedrobcst/Xerus/releases/tag/v1.0r1>
- 作用：从开放结构数据库获得候选结构，GSAS-II 模拟单相谱，使用相似度初筛，并通过快速 Rietveld 精修与迭代去峰进行多相识别；精修可输出 weight fraction。
- 当前状态：旧内嵌 GSAS-II 的 Python 3.8/x86_64 二进制仍不兼容；现已改用官方
  GSAS2MAIN Python 3.13/arm64 环境和 `backcompat` 导入层。3 个本地 CIF 模拟已跑通，
  固定随机种子的 50/50 MgO/TiO₂ 已知候选精修得到 Rwp=8.08%、质量分数 0.49395/0.50605。
  默认 MongoDB/在线结构库驱动的盲搜索尚未执行；这一步才对应 v2 主测试。
- 原生输入：XRD、元素列表、GSAS-II instrument parameter file；不需要真实逐样本相数，
  可统一用 `n_runs=3` 或预注册的 `auto`。
- 安装提示：论文仓库较旧并捆绑历史 GSAS-II。新机器优先安装官方 GSAS2MAIN，再把
  XERUS 源码作为前端；需要 MongoDB，并按仓库说明配置 Materials Project API key。

### 2. Dara

- 类型：Python 包、命令行工具和本地 Web 服务；精修后端为 BGMN。
- Python 包名：`dara-xrd`。
- 文章：Yuxing Fei, Matthew J. McDermott, Christopher L. Rom, Shilong Wang, Gerbrand Ceder, “Dara: Automated Multiple-Hypothesis Phase Identification and Refinement from Powder X-ray Diffraction”.
- 年份与期刊：2026，*Chemistry of Materials* 38, 1364-1376。
- DOI：<https://doi.org/10.1021/acs.chemmater.5c02820>
- 代码：<https://github.com/CederGroupHub/dara>
- 文档：<https://CederGroupHub.github.io/dara/>
- PyPI：<https://pypi.org/project/dara-xrd/>
- 作用：在候选化学空间中生成多套物相组合假设，调用 BGMN 逐层进行全谱 Rietveld 验证和排序，并返回 BGMN 的 `GEWICHT` weight fraction。
- 当前状态：Dara 1.3.0 已在项目隔离环境中运行；本地 CIF 的候选相条件精修已跑通。
  COD 原生数据库下载与 100 条端到端搜索尚未完成。
- 原生输入：XRD、元素集合、BGMN instrument profile 和结构数据库。源码默认可用 COD；
  ICSD 是商业数据库，若使用必须单独声明，不能与 COD 结果混写。
- 最小安装：`python -m venv dara-env && dara-env/bin/pip install dara-xrd==1.3.0`。
  Linux 上若随包 BGMN 与 GLIBC 不兼容，按 Dara 文档安装 BGMN 后设置其可执行路径。
  下载完整 COD 会较慢；也可让 Dara 按每条元素集合从 COD 在线下载过滤后的 CIF 并缓存。

### 3. CrystalShift + CrystalTree

- 类型：Julia 包，不是 Python 库。
- 文章：Ming-Chiang Chang et al., “Probabilistic phase labeling and lattice refinement for autonomous materials research”.
- 年份与期刊：2025，*npj Computational Materials*。
- DOI：<https://doi.org/10.1038/s41524-025-01627-0>
- CrystalShift 代码：<https://github.com/MingChiangChang/CrystalShift.jl>
- CrystalTree 代码：<https://github.com/MingChiangChang/CrystalTree.jl>
- 作用：CrystalShift 优化晶格参数、峰宽、背景和 phase activation；CrystalTree 通过 best-first tree search 和 Bayesian model comparison 搜索物相组合。
- 定量限制：论文明确说明当前 peak amplitudes 没有像完整 Rietveld refinement 那样与晶体结构因子严格关联。因此源码中的 `get_fraction()` 或 `get_moles()` 不能未经额外校准就当作质量分数或真实摩尔分数。
- 当前状态：已用项目内 Julia 1.12.6 运行已知候选拟合和三候选相搜索；由于旧依赖存在预编译警告，当前运行命令使用 `--compiled-modules=no`。
- 关键输入限制：CrystalShift/CrystalTree 不根据元素自动查数据库。必须由用户先准备候选
  CIF，所以 v2 主结果必须写成例如 `CrystalShift + COD front-end`；单独写
  `CrystalShift` 会把额外检索步骤隐藏掉。
- 安装提示：安装 Julia 后新建独立 project，并运行：

```julia
using Pkg
Pkg.add(url="https://github.com/MingChiangChang/CrystalShift.jl")
Pkg.add(url="https://github.com/MingChiangChang/CrystalTree.jl")
Pkg.add("JSON")
```

  CIF 预计算脚本还需要独立 Python 环境中的 `pymatgen`、`numpy` 和 `xrayutilities`。
  所有样本统一 tree depth 3，不按真值分别设 2 或 3。

## v2 方法输入边界

| 方法 | 只给 XRD + 元素能否原生运行 | 数据库/候选来源 | 是否需要逐样本真实相数 |
|---|---|---|---|
| MatDiffract | 能 | Atomly 完整索引 | 否；统一最多 3 相 |
| XERUS | 能 | MP/OQMD/AFLOW/COD + 本地缓存 | 否；`auto` 或统一 3 runs |
| Dara | 能 | COD；也支持用户 ICSD/自定义库 | 否；统一 `max_phases=3` |
| CrystalShift + CrystalTree | 不能 | 用户预先提供 CIF；需外接并声明数据库前端 | 否；统一 depth 3 |

## 支撑软件

### pymatgen

- 类型：Python 材料分析库。
- 官网与文档：<https://pymatgen.org/>
- XRD API：<https://pymatgen.org/pymatgen.analysis.diffraction.html>
- 代码：<https://github.com/materialsproject/pymatgen>
- 本项目用途：从单个 CIF 计算理想运动学衍射棒图。它不提供“给定质量分数后自动生成多相定量谱”的高级接口。
- 关键点：`scaled=True` 会把每个单相的最大峰设为 100；构造多相谱必须从 `scaled=False` 开始，并在外部施加具有物理定义的相 scale factor。

### GSAS-II

- 类型：Python 开源晶体学与 Rietveld 软件，也是 XERUS 的后端。
- 官网：<https://advancedphotonsource.github.io/GSAS-II-tutorials/>
- 代码：<https://github.com/AdvancedPhotonSource/GSAS-II>
- 仪器 profile 教程：<https://advancedphotonsource.github.io/GSAS-II-tutorials/CWInstDemo/FindProfParamCW.html>
- 本项目用途：候选相的完整 Rietveld 精修、仪器 profile 建模和质量分数反算验证。

### BGMN

- 类型：独立 Rietveld 精修程序，不是 Python 库；Dara 通过 Python 调用它。
- 官网：<https://www.bgmn.de/>
- 本项目用途：Dara 的全谱拟合与 weight fraction 输出。

XERUS、Dara、CrystalShift、Hill-Howard QPA 和 GSAS-II 的上述主要方法论文均为同行评审期刊论文；本清单没有把预印本或综述当作这些软件的主要方法依据。
